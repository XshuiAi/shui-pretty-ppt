01. 研究问题

为什么 PPT 正在转向 HTML 演示？

当内容来自 Markdown、飞书文档、图片、视频和表格时，HTML 更容易承载长内容、交互结构、在线分享与持续迭代。

  • 传统 PPT 更适合固定页数，但修改和复用成本高。
  • HTML 可以自然支持滚动、点击展开、视频嵌入和数据动效。
  • 同一份结构可以沉淀为模板，下一次直接套用。

第一阶段：内容收集

整理原始文档、图片、表格与口头补充，标记必须展示和可放附录的内容。

第二阶段：结构成稿

把长内容拆为页面，优先保留论证链路，避免把所有文字塞进同一页。

第三阶段：视觉校准

统一蓝白配色、标题栏、底部波浪和内容卡样式，保证正式感和辨识度。

第四阶段：现场演示

通过滚动、箭头和逐块显现控制节奏，减少误触跳页带来的风险。

结论一

HTML 更适合承载复合型内容

当资料里同时出现文字、图片、视频、表格和网页链接时，HTML 比单页 PPT 更容易组织为可浏览、可跳转的结构。

  • 章节导航清晰
  • 长内容可分层展开
  • 可直接部署分享
结论二

正式场合要克制动效

宝蓝答辩稿的动效重点是逐块显现、章节定位和图表进入，不做过度花哨的装饰动画。

  • 点击显现而不误跳页
  • 滚动和箭头负责翻页
  • 视觉重心始终在内容上
