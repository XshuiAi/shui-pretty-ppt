# Ribbon Tab Brochure（彩签页报）

Use this style for full-screen project brochures, product explainers, location/project decks, operations reports, service packages, and lightweight external proposals that need to feel like a browsable website rather than a traditional slide deck.

## Best For

- project brochures, real-estate or space introductions, campus/venue guides
- product or service package introductions
- operations reviews with metrics and progress highlights
- business proposals that need a friendly but organized visual system
- document-to-HTML pages where the user wants clickable navigation and full-screen presentation behavior

## Visual Language

- full-screen slide pages with no outer gallery frame
- top colored ribbon tabs inspired by website/category navigation
- black fine-line brochure framing inside each page
- off-white and pale pastel section backgrounds: light blue, pink, mint, yellow, and dark closing page
- structured left content + right data panels
- large numbers, data rows, map-like diagrams, card walls, and plan-list modules
- restrained scrapbook/card collage elements for emphasis, without becoming too childish

## Layout Grammar

- Use a full-screen deck rhythm: cover, overview, location/data, plan list, card wall, metrics, use cases, closing.
- Keep every desktop slide designed to fit within one viewport.
- Use top tabs for fast navigation instead of a heavy sidebar.
- Use black line frames and data panels to keep pastel colors grounded.
- Use cards and stickers sparingly, mainly for secondary notes or highlights.
- For formal content, increase white/off-white space and reduce sticker density.

## Interaction Grammar

Interactions should feel like a browsable brochure:

- colored tabs jump to sections and update active state on scroll
- select menu for precise page jumps
- blank slide clicks do not navigate; wheel scrolling, bottom arrows, keyboard arrows, tabs, and the select menu handle page movement
- plan rows update a detail card
- card wall hover lift
- data bars animate when the metrics page enters
- reduced-motion support

Avoid noisy sticker overload, excessive rotations, bouncy animations, or reusing reference images directly.

## Content Mapping

When adapting real user content:

- project / brand name -> cover title and top logo area
- sections / categories -> ribbon tabs
- project images -> architecture/image-window page
- core metrics -> right data panel or metrics page
- location/market/user distribution -> map-style page
- product/service modules -> clickable plan list
- examples/screenshots/notes -> card wall
- use cases -> four-card scenario grid
- call to action -> dark closing page

## Common Page Types

- Full-screen ribbon cover
- Project overview with data panel
- Location / positioning page
- Clickable plan list page
- Collage-style highlight card wall
- Operations metrics dashboard
- Use-case grid
- Dark closing / call-to-action page
