# Ivory Research Deck（象牙研稿）

Use this style for academic, research, workplace, and formal explanation pages that need more information density than a light visual story, while still feeling designed and readable.

## Best For

- academic talks, literature reviews, thesis-style summaries
- workplace reports, quarterly reviews, strategy briefs
- product research, competitive analysis, user insight reports
- course notes, lecture pages, serious explainers
- Markdown or document-to-HTML presentations with charts, tables, images, and video modules

## Visual Language

- grey exhibition-like page background
- ivory paper slide surfaces with subtle ruled-paper texture
- compact uppercase navigation labels such as `ASPIRATION`, `INFODATA`, `TARGET`
- strong black rules, stacked horizontal lines, and black footer bands
- pale blue used sparingly for small labels, thin emphasis, or one secondary panel; the dominant impression should be black, ivory, and grey
- thin tables, grids, timelines, and research-card layouts
- minimal portrait/avatar marks when a speaker, persona, or stakeholder group is useful

## Layout Grammar

- Treat each section as a presentation board, not a normal web card.
- Use one major idea per board, but allow denser evidence inside tables, grids, or timelines.
- Use pale blue only for information emphasis; avoid large turquoise blocks on the cover.
- Use black bars for final takeaways, section labels, or closing emphasis. If a black bar contains an arrow, make it a real clickable control.
- Use uppercase micro labels and page markers to create academic/report structure.
- Use wide margins and thin borders so the page feels formal, not like a dashboard.

## Interaction Grammar

Interactions should be calm and functional:

- scroll reveal for each board
- line/rule drawing on page entrance
- quick page jump selector for long reports
- clickable metric cards with explanatory notes
- expandable evidence or speaker-note cards
- hover/focus typewriter reveal for short author notes, research hints, or evidence captions
- black footer bars can scroll to the next board when clicked
- tabs for use cases, stakeholder groups, or research perspectives
- reduced-motion support

Avoid playful sticker motion, loud gradients, glowing effects, heavy lift-on-hover cards, or decorative animation that distracts from the report.

## Content Mapping

When adapting real user content:

- title or research question -> cover
- executive summary or abstract -> abstract board
- table of contents -> contents board
- evidence, screenshots, data, or images -> table/media boards
- argument progression -> timeline or axis board
- methods, workflow, or process -> step grid
- recommendations -> black footer band or closing board
- long evidence -> expandable detail cards

## Common Page Types

- Cover board
- Contents board
- Abstract / executive summary
- Problem framing
- Timeline / argument shift
- Feature or evidence cards
- Comparison table
- Interaction or method demo
- Use-case tabs
- Workflow steps
- Closing / conclusion board
